# Synapse v0.2 — Observability + Safety Layer for Any Multi-Agent Stack

**Status:** Planning · drafted 2026-05-07
**Owner:** Aadit Rajgor
**Target:** v0.2.0 release — 5 weeks of focused work

This roadmap is the result of pressure-testing v0.1's positioning against five real multi-agent personas (solo dev with Claude Code + Cursor; CrewAI research crew; parallel code reviewers; engineer team with multiple Claude Code sessions; production agent stack with billing + tech-support). The conclusion: v0.1 is a useful coordination *primitive*, not yet a complete coordination *layer*. v0.2 closes that gap.

The honest distillation that drives this plan:

> **Synapse is the missing observability + safety layer for any multi-agent stack, with conflict detection as the headline safety feature.**
>
> Not a coordination protocol that happens to come with a dashboard.

---

## What v0.1 revealed

Two realistic 4-agent workloads (Instagram-clone backend, data-analysis pipeline — see `runtime/modal/_payloads/real_app_*.py`) running real Anthropic Haiku 4.5 calls in Modal sandboxes exposed three findings:

1. **The original L2 router only caught simultaneous collisions.** Sequential same-resource overwrites (the most common multi-agent failure mode) slipped through. Fixed in commit `217e7a7` with a `resolved_lookback_ms` window and the new `stale_base_overwrite` conflict kind. Without this fix Synapse caught 0/5 expected collisions; with it, 5/5.
2. **Catching ≠ solving.** Every CONFLICT we routed was logged and ignored — the collision still happened. The integrations had no concept of "what to do about it."
3. **Detection is structural, not semantic.** Two agents writing different scope names that mean the same thing semantically (e.g. `cleaner.revenue_formula = qty*price` vs `analyst.revenue_formula = qty*price*(1-discount)` written to different files) slip through entirely. The coordinator already does belief-divergence detection, but it isn't on the integration path.

v0.2 addresses 2 and 3 directly, and re-positions the whole project around the audit + observability story that v0.1 buried.

## Strategic decisions (locked)

| Decision | Rationale |
|---|---|
| **Envelopes are canonical** | INTENTION/CONFLICT/RESOLUTION/BELIEF carry semantics OTel doesn't (`blocking`, `gate_ms`, `expected_outcome`, `confidence`, `evidence`). OTel is one I/O format among several. |
| **Bring-your-own-LLM** | The user's agents already have an LLM. Synapse must never make a paid LLM call without explicit caller config. Internal reasoning (scope inference, belief divergence, auto-merge) reuses the user's existing model. If `set_llm()` is never called, Synapse runs in rules-only mode and gracefully degrades. |
| **Audit-first adoption** | Read existing traces before writing live. Every modern agent framework already emits traces (OTel / LangSmith / W&B Weave / Phoenix / OpenInference / JSONL). Auditing them retroactively gives framework-agnostic value with zero integration code. |
| **OpenInference-flavored OTel as substrate** | Vendor-neutral, increasingly default, Arize/Phoenix already standardized tool-call span attributes. LangSmith and JSONL are extra importers. |
| **Universal SDK + opt-in framework adapters** | One `synapse.intend()` context manager + auto-detect for OTel-emitting frameworks. Per-framework wrappers stay as sugar. Don't ship N integrations when one normalizer covers all. |

These are recorded formally in `spec/adr/ADR-0003-byo-llm-and-audit-first.md`.

## Changes by component

### Adding

| Component | What | Why |
|---|---|---|
| `sdk-python/synapse/audit/` | Trace importers (OTel/OpenInference, LangSmith, JSONL), scope-inference engine, HTML/JSON report generator | The wedge: works with any framework that emits any common trace format. Read-only entry point converts skeptics in 60s with their own data. |
| `sdk-python/synapse/llm/` | `synapse.set_llm()`, `from_anthropic()`, `from_openai()`, `from_langchain()`, `from_litellm()`, `auto()` | BYO-LLM: caller passes their existing client; Synapse uses it for scope inference, belief divergence, auto-merge. Never charges anyone's account without explicit consent. |
| `sdk-python/synapse/intend.py` | `synapse.intend(scope, agent, ...)` async context manager | Universal SDK — works in any Python codebase regardless of framework. Replaces the per-framework wrapper as the recommended path. |
| `sdk-python/synapse/install.py` | `synapse.install()`, `synapse.install(framework="langgraph"\|"crewai"\|"autogen"\|...)`, `synapse.install(otlp_endpoint=...)` | One-line auto-instrumentation. OTel-emitting frameworks get Synapse for free; explicit framework adapters as opt-in sugar. |
| `runtime/cli/` | `synapse audit`, `synapse up`, `synapse down`, `synapse install`, `synapse demo` | 30-second adoption requires CLI parity with `docker` / `pip`, not docs. |
| `runtime/policies/` | `MergePolicy` ABC + four built-ins: `redirect` (default), `wait`, `abort`, `auto_merge` | Catching ≠ solving. Policies are the bridge from "we detected a CONFLICT" to "the multi-agent system actually does the right thing." |
| `sdk-typescript/src/{audit,intend,install,llm}/` | TS mirrors of all the above | Polyglot stacks need polyglot SDKs. Same surface, same shapes. |
| `docs/{quickstart,llm-config,audit,policies,beliefs}.md` | New docs centered on the new entry points | Re-pitch from "protocol" to "observability + safety layer." |

### Changing

| Component | What | Why |
|---|---|---|
| `runtime/router/worker.py` | After today's `stale_base_overwrite` fix: add MergePolicy hook — when CONFLICT fires, dispatch to registered policy first; only route to inbox if policy returns `inbox` | Today the router only fans out warnings. Policy hook lets it actively coordinate (block, redirect, merge). |
| `runtime/coordinator/agent.py` | BELIEF divergence runs on every BELIEF emission (not 30s tick). Divergence signals route directly to other agents' inboxes. | Persona-2 (research crew with contradictory facts) needs sub-second feedback; 30s is useless for live coordination. |
| `sdk-python/synapse/integrations/hermes_integration.py` | Refactor internals to use `synapse.intend()` context manager. Public API unchanged. | New users use `intend()` directly; existing users with `wrap_tool_call_for_synapse` keep working. |
| `sdk-typescript/src/integrations/{paperclip,openclaw}.ts` | Same refactor — internals call `synapse.intend()` | Same reason. Backward compat. |
| `runtime/router/l3_semantic.py` | Read LLM from `synapse.set_llm()` instead of constructor arg | Routes consolidate around one config point; no more per-component LLM wiring. |
| `README.md` + landing page | Re-pitch: "the missing observability + safety layer for any multi-agent stack" | Position. Conflict detection is the headline feature, not the entire pitch. |

### Removing

| Component | What | Why |
|---|---|---|
| `runtime/coordinator/` Gemini default backend | Hard-coded model defaults | All LLM use goes through `synapse.set_llm()`. No backend gets a "default" — caller decides. If unset, Synapse runs in rules-only mode and skips LLM-mediated paths. |
| Inline `os.environ["ANTHROPIC_API_KEY"]` reads inside coordinator | API key reading from env directly | BYO-LLM means caller hands us a configured client, not a key. Removes a class of "why didn't it work" bugs. |

Nothing else gets removed — we're additive on the protocol surface and the existing per-framework wrappers all stay as sugar over the new universal SDK.

## The new public API surface

Three entry points, in order of laziness:

```python
# 1. Laziest — auto-detect everything
import synapse
synapse.install()  # detects OTel, picks up your LLM from env, starts the dashboard

# 2. Medium — explicit framework + LLM
import synapse
from anthropic import AsyncAnthropic
synapse.install(
    framework="langgraph",
    llm=synapse.from_anthropic(AsyncAnthropic()),
    merge_policy=synapse.MergePolicy.redirect,
    critical_scopes=["billing.*", "prod.deploy.*"],
)

# 3. Most explicit — manual context-manager API
import synapse
async with synapse.intend(
    scope="repo.fs.auth.py:w",
    agent="code-reviewer",
    expected_outcome="fix CVE-2026-1234",
) as i:
    if i.conflicts:
        # caller decides: pivot, wait, redirect, abort
        await i.pivot()
    result = await my_tool_call()
    i.resolve(state_diff={"lines_changed": 47})
```

CLI:

```
synapse up                       # Docker Compose: Redis + Postgres + router + UI
synapse audit ./traces.jsonl     # Read-only conflict report from any trace export
synapse audit --langsmith URL    # Pull from LangSmith API directly
synapse install --framework=langgraph  # Print copy-paste install snippet
synapse demo                     # Spawn a multi-agent demo workload locally
synapse down                     # Tear down
```

## BYO-LLM design

The right shape isn't one LLM — it's two roles, with a smart default:

```python
synapse.set_llm(
    primary=user_llm,                   # what their agents already use; for user-facing decisions (auto-merge)
    internal=user_llm.cheap_variant(),  # for high-frequency internal reasoning (scope inference, belief divergence)
                                        # defaults to the cheapest model in the same provider family
)
```

If the user just calls `synapse.set_llm(client)`, Synapse picks a "cheap variant" automatically:

| Primary | Auto-selected internal |
|---|---|
| Anthropic `claude-sonnet-*` / `claude-opus-*` | `claude-haiku-*` (cheapest in family) |
| OpenAI `gpt-4o` / `o1-*` | `gpt-4o-mini` |
| Ollama `llama3.1:70b` | `llama3.1:8b` (if installed locally) |
| Gemini `gemini-2.5-pro` | `gemini-2.5-flash` |

Bridges for popular ecosystems (using existing v0.1 adapters as the substrate):

```python
# LangChain
from langchain_anthropic import ChatAnthropic
synapse.set_llm(synapse.from_langchain(ChatAnthropic(model="...")))

# LiteLLM (universal across 100+ providers)
import litellm
synapse.set_llm(synapse.from_litellm(model="anthropic/claude-haiku-4-5"))

# Local Ollama
synapse.set_llm(synapse.OllamaAdapter(host="http://localhost:11434", model="llama3.1:8b"))

# vLLM self-hosted
synapse.set_llm(synapse.OpenAIAdapter(base_url="http://localhost:8000/v1", model="..."))
```

**Critical guarantee:** if `set_llm()` is never called, Synapse runs in rules-only mode and gracefully degrades. L1 router (rules) and L2 router (SQL conflict detection) still work; L3 (semantic), belief divergence, scope inference fallback, and auto-merge become no-ops with clear log messages. No surprise charges on anyone's account.

## Real product-dev demo gallery

Each lives under `runtime/modal/_payloads/`, mirroring the structure of the existing `real_app_instagram.py`. They become the demo gallery on the landing page.

### Demo 1 — Instagram-clone backend, with auto-merge

Already built (`real_app_instagram.py`). v0.2 extension:

- 4 agents: `db_engineer`, `api_engineer`, `auth_engineer`, `feed_engineer`
- Same 12 file writes, same overlaps on `models/user.py` (3-way) + `api/posts.py` (2-way)
- New: `MergePolicy.redirect` configured on `repo.fs.models/*:w`
- Flow when `api_engineer`'s `models/user.py` write fires CONFLICT:
  1. Policy reads `db_engineer`'s RESOLUTION state_diff (the file content as written)
  2. Policy injects into `api_engineer`'s next LLM call: "db_engineer wrote this file 12s ago. Here's their version. Re-write yours to incorporate their fields without losing yours."
  3. `api_engineer` re-emits the INTENTION (now flagged as a merge attempt)
  4. The file ends up with `id, username, email, created_at` (db) + `bio, avatar_url` (api) + `password_hash, last_login` (auth)
- **Headline result:** `git diff` between no-Synapse run and with-Synapse run. The no-Synapse `models/user.py` has only auth's fields; the Synapse run has all eight.

### Demo 2 — E-commerce checkout flow (selective enforcement on real money)

- 4 agents: `cart_engineer`, `payment_engineer`, `shipping_engineer`, `fraud_engineer`
- They build: `cart_service.py`, `payment_processor.py`, `shipping_calc.py`, `fraud_check.py`
- Shared models: `models/order.py` (cart + payment + shipping all touch), `models/customer.py` (cart + fraud)
- `critical_scopes=["repo.fs.payment_processor.py:w", "billing.*"]` configured
- Demonstrates: when `payment_engineer` attempts to modify `payment_processor.py` after `fraud_engineer` flagged a transaction, the intention is **hard-blocked** with `MergePolicy.abort`, and the agent's framework gets back a clear AdapterError it can surface to the human-in-the-loop
- **Headline result:** an audit trail showing every payment-related intention, who attempted it, who blocked it, with sub-second decisions. The production safety story.

### Demo 3 — Marketing website (BELIEF divergence catches what scope can't)

- 4 agents: `designer`, `copywriter`, `frontend_dev`, `seo_specialist`
- They build: `design_tokens.json`, `copy.md`, `components/`, `sitemap.xml`
- Almost zero scope overlap — different files
- The collisions are semantic:
  - `designer` BELIEF: `brand_color = "#FF5733"`
  - `frontend_dev` BELIEF: `brand_color = "#FF6F61"` (off-brand drift)
  - `copywriter` BELIEF: `tone = "formal"`
  - `seo_specialist` BELIEF: `tone = "conversational"` (better for engagement)
- BYO-LLM does belief-divergence detection on every emission
- Synapse catches both disagreements *before* the inconsistent artifacts ship
- **Headline result:** rendered website screenshot from no-Synapse run (off-brand color, mixed-tone copy) vs Synapse run (consistent palette, consistent voice). The demo for non-coding multi-agent work.

### Demo 4 — Data analysis pipeline, extended with BELIEFs

- The existing `real_app_data_analysis.py` workload
- Extension: each agent auto-emits BELIEFs from tool results via `synapse.install(emit_beliefs_from_tool_results=True)`
- `data_cleaner` writes the rename code, BELIEF auto-emitted: `column_revenue_formula = "qty * unit_price"`
- `analyst` writes the analysis, BELIEF auto-emitted: `column_revenue_formula = "qty * unit_price * (1 - discount)"`
- Coordinator runs LLM-judged divergence detection (using user's `set_llm`) within ~500ms
- `analyst`'s next intention gets a system reminder in their inbox: "data_cleaner believes revenue = qty\*price (no discount); your formula will produce different numbers. Reconcile before proceeding."
- **Headline result:** the revenue numbers in the final report match across cleaner's pre-aggregations and analyst's roll-ups.

### Demo 5 — Stress test: 8 agents on a real OSS bug backlog

- Pull a public OSS repo with N open bugs
- Spawn 8 LangGraph crews, each tasked with a different bug, all touching the same codebase
- Run with `synapse.install()` and audit + live mode both on
- Measure: PR success rate (bugs closed cleanly with no rework) with vs without Synapse
- This is the **empirical proof** for the launch announcement.

## Open questions worth resolving before week 1 PRs land

1. **Normalized event schema shape.** Internally we keep envelopes; for the audit input, what's the minimal event record? Proposed:
   ```
   AuditEvent {
     trace_id, span_id, parent_span_id,
     agent_id, session_id, tool_name, tool_args,
     ts_start_ms, ts_end_ms,
     scope_inferred: list[str],   # filled by scope_inference.py
     status: "ok" | "error",
     state_diff?: dict,           # if known
     raw: dict                    # original importer payload for round-tripping
   }
   ```
2. **Where does the audit run conflict detection?** Two options:
   - In-process Python — uses an in-memory SQLite or DuckDB to host the events table and run the same SQL the L2 router does
   - Against a real Postgres — requires `synapse up` first
   Lean toward in-process SQLite/DuckDB for the audit so it runs without infra. Conflict-detection SQL is portable.
3. **Auto-emit BELIEFs from tool results — opt-in or default?** Lean opt-in (`emit_beliefs_from_tool_results=True`). Default-on risks surprising users with extra LLM calls.
4. **Hosted Synapse — paid convenience for prototyping?** Defer past v0.2. Focus on self-hosted flow first.

## What this roadmap commits us to *not* doing

- Shipping more conflict kinds beyond what's in v0.1
- Building a full UI redesign before the audit/install loop is proven
- Chasing enterprise features (SSO, audit logs, RBAC) before product-market fit
- Trying to be the agent framework — Synapse wraps frameworks, never replaces them
