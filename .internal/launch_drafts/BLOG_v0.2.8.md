# Ten Agentic SDKs Just Built a Working Flask App Together. Here's the Audit Log.

*~1100 words. Skeleton — edit voice, examples, and CTAs before publishing.*

---

There's a question quietly killing every enterprise that's tried to deploy
agentic teams in production:

> When ten different vendor SDKs collaborate on one task, how do you prove
> who did what?

Today I'm releasing **Synapse v0.2.8** — and with it, a bundle that answers
that question concretely: a Flask Todo app built end-to-end by ten different
agentic framework runtimes, with one unified audit log.

The bundle is committed. The app runs. The audit trail is real.

## The setup

I gave each of these ten SDKs a different role in building one Flask app:

| Vendor SDK | Role | File it wrote |
|---|---|---|
| Microsoft AutoGen | API Architect | `api_spec.md` |
| CrewAI | Backend Engineer | `main.py` (the actual Flask app) |
| LangChain LangGraph | Test Writer | `test_app.py` |
| Synapse-native Hermes | Project Coordinator | `PLAN.md` |
| HuggingFace smolagents | DB Modeler | `models.py` |
| Agno | Docs Writer | `README.md` |
| LlamaIndex | Lint Reviewer | `LINT.md` |
| Pydantic AI | Schema Validator | `schemas.py` |
| OpenAI Agents SDK | Deploy Engineer | `deploy.sh` |
| Google ADK | Final Reviewer | `REVIEW.md` |

Each ran inside its own framework's runtime — its own tool-dispatch loop,
its own LLM call pattern, its own conventions. The only thing they shared
was a single Synapse session.

## What Synapse does in this picture

Synapse is **not** another agent framework. It's the layer **above** the
frameworks — a thin envelope protocol that captures three things from any
agent, regardless of which SDK the agent runs on:

1. **INTENTION** — "I'm about to do X on scope Y"
2. **THOUGHT** — "Here's what I'm reasoning right now" (captures native
   thinking blocks from Anthropic / o-series, falls back to message
   content as PSEUDO_THOUGHT)
3. **RESOLUTION / CONFLICT / PIVOT / BELIEF / COST_REPORT** — what
   happened next

Every adapter installs into its host framework at the tool-dispatch path
(`function_tool`, `BaseTool.invoke`, `AgentWorkflow._call_tool`, etc.) and
emits envelopes into a shared Postgres + Redis pair. **One session, ten
runtimes, one log.**

## What landed in v0.2.8

The headline:

> **10/10 framework adapters PASSed deterministically, byte-for-byte
> reproducible across runs**, and the cooperative-build demo above produced
> a Flask app that actually serves HTTP on port 5001.

The proof, reproducible in 10 seconds after a clone:

```bash
git clone https://github.com/arajgor1/synapse && cd synapse
pip install flask
cd bench/results/v32_app_bundle
python -c "import main; print(main.app.test_client().get('/todos').status_code)"
# → 200
```

That `main.py` file isn't sample code I wrote. It was generated by the
CrewAI agent under the role of Backend Engineer, then validated by my
fallback layer (because gpt-4o-mini originally passed only a one-line
summary instead of the full source — more on that below), then committed
verbatim to the repo. You can read it: 12 lines, two routes,
one in-memory list.

## What's hard and what's still hard

**What was hard but solved**:

- **Per-task agent attribution under `asyncio.gather`**: env-var
  agent IDs raced. Replaced with `ContextVar` (`synapse.agent_context`).
- **LangGraph's nested ToolNode dispatch**: the top-level
  `RunnableConfig.callbacks` doesn't propagate into nested toolcalls.
  Fix: `register_configure_hook(inheritable=True)`.
- **llama-index-core ≥0.11's Workflow rewrite**: the old `ReActAgent.chat()`
  is gone. The canonical tool-dispatch path is now
  `BaseWorkflowAgent._call_tool` and `AgentWorkflow._call_tool`.
- **THOUGHT envelopes silently dropped**: `Bus` exposes
  `publish_session()` and `publish_inbox()`, never `publish()`. The
  Anthropic wrapper called `publish()`, the AttributeError was caught at
  debug-level, every THOUGHT was discarded silently for three iterations
  before I found it. The fix is one line + an upgrade to warning-level
  logging on emit failures.

**What's still hard** (carrying forward to v0.2.9):

- **gpt-4o-mini dispatches tools with empty content args** in 3 of 10
  frameworks (LangGraph, smolagents, Agno). The model says "I'll call
  write_artifact" then calls it with `content=""`. The fallback layer
  rescues the artifact via a direct LLM call, but no INTENT is registered
  for that agent. Anthropic Sonnet / Haiku don't have this issue.
- **Reproducibility under real wall-clock**: the convergence bench is
  deterministic in agent counts and intent counts, but real wall-clock
  fluctuates 8-12% based on network latency to the LLM provider.

## What you can do with it right now

1. **Pip install it**: `pip install synapse-protocol`
2. **Audit existing traces**: `synapse audit ./your-traces.json` — supports
   OpenInference, LangSmith, Bedrock, Vertex, Azure, plain JSONL
3. **Drop into your existing stack**: `synapse.install(framework="autogen")`
   and your agents start emitting envelopes; no code changes needed
4. **Reproduce the v32 cooperative build**: instructions above
